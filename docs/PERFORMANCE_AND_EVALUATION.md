## 2. 任务一清洗指标

任务一返回的质量报告包含文件数、格式分布、字符变化、解析错误、空文本、残留噪声、重复内容、术语替换和清洗耗时。在线演示以 DataMate 任务结果和 MCP 工具返回为准，不补写工具没有返回的语义噪声明细。

标准验证数据集为“糖尿病全流程演示数据集”，覆盖 `txt/csv/json/jsonl` 四种格式。验收时应确认最终数据集仍保留四种源格式，而不是被合并为单一文本格式。

## 3. 任务二抽取与图谱构建指标

任务二返回文件解析、实体识别、关系抽取、三元组生成、三元组入库和分析库刷新等阶段指标。可见指标包括：

| 指标 | 展示口径 |
| --- | --- |
| 吞吐量 | 处理记录数 / 抽取耗时 |
| 平均延迟 | 抽取耗时 / 处理记录数 |
| 入库耗时 | 三元组写入与去重阶段耗时 |
| 结果质量 | 实体数、关系数、生成三元组数、入库三元组数、过滤原因 |

实体识别基线可通过 `scripts/evaluate_task2_cmeee_f1.py` 复现。关系抽取自检可通过 `scripts/evaluate_task2_cmeie_f1.py` 复现。外部 CBLUE 数据不随工程目录内置时，请使用 `--root` 或 `CCF_EVAL_DATA_ROOT` 指定数据目录。

## 4. 任务三 NL2SQL 指标

| 指标 | 当前值 |
| --- | --- |

复现命令：

```bash
python scripts/evaluate_task3_nl2sql_templates.py
```

该命令读取 `tests/task3_nl2sql_eval_set.json` 与 `data/task3_analytics.db`，输出 JSON 报告并更新本说明中的 NL2SQL 指标文档。

## 5. NPU 边界

当前在线环境未检测到可用于本项目的真实 NPU 运行时，因此不声明 NPU 加速和 CPU/NPU 对比提升。若后续迁移到具备 NPU 的环境，应单独补充吞吐量、延迟和资源利用率对比。
