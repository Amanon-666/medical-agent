# 数据资产说明

本项目的数据资产分为三类：任务输入数据、知识图谱数据库、分析数据库。源码和数据库分开管理，数据库作为独立数据产物随工程目录提供。

## 1. 任务输入数据

```text
data/standard_diabetes_demo/datamate_upload/
├── 糖尿病医患问答脏文本.txt
├── 糖尿病病例表格.csv
├── 糖尿病知识图谱子集.json
└── 糖尿病医疗记录.jsonl
```

该目录保留糖尿病混合格式数据集，覆盖 `txt/csv/json/jsonl`。数据中包含受控噪声和格式问题，例如 HTML 片段、URL、Emoji、繁体字、全角字符、HIS 导出头、测试标记和医学缩写，用于展示任务一清洗算子的实际效果。

## 2. 知识图谱数据库

| 文件 | 说明 |
| --- | --- |
| `data/task2_medical_kg.db` | 任务二知识图谱数据库，包含实体、关系、三元组、来源和质量审计记录。 |

该数据库由任务二流水线写入，任务三关系子图和数据来源列表也读取该库。

## 3. 分析数据库

| 文件 | 说明 |
| --- | --- |
| `data/task3_analytics.db` | 任务三分析数据库，包含疾病、症状、药物、检查、科室、并发症等扁平统计表。 |

该数据库服务于疾病详情问答、NL2SQL、统计图表和证据表。

## 4. 数据使用边界

- 数据库文件不建议纳入 Git 历史，可作为独立数据文件随工程目录提供。
- 在线 DataMate 中已注册的糖尿病混合格式数据集与本目录中的输入数据保持同类结构。
- 任务一结果应保持源文件格式和文件数，不应把混合格式数据统一压成纯文本。
- 测试过程中新增的数据来源不属于基础数据资产；需要清理时应在可视化平台或数据库维护工具中按来源删除。
