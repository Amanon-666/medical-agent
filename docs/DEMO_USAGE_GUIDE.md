# 在线服务操作说明

本文说明如何使用已经部署好的医学数据智能体服务。所有入口均连接同一套在线环境，适合快速查看任务一、任务二、任务三的完整流程。

## 1. 服务入口

| 服务 | 地址 | 说明 |
| --- | --- | --- |
| 医学数据智能体可视化平台 | `https://demo.mashiro.xin/` | 展示数据来源、关系子图、统计图表、证据表、噪声拦截和疾病问答。 |
| Nexent 智能体平台 | `https://nexent.mashiro.xin/` | 与任务一、任务二、任务三智能体对话。 |
| DataMate 数据处理平台 | `https://datamate.mashiro.xin/` | 查看混合数据集、算子链、清洗任务和输出数据集。 |

演示账号：

| 平台 | 账号 | 密码 |
| --- | --- | --- |
| Nexent / DataMate | `suadmin@nexent.com` | `241002814` |

## 2. 可视化平台操作

打开 `https://demo.mashiro.xin/` 后，应能看到三个工作区：

1. **数据流向**：展示原始混合数据、任务一清洗结果、任务二知识图谱来源和任务三分析库状态。
2. **自然语言问答**：输入疾病相关问题，系统返回结构化结果、证据和图表。
3. **图谱洞察**：查看疾病关系子图、统计图表、证据表和噪声拦截记录。

可使用以下问题验证页面：

```text
肺泡蛋白质沉积症有哪些症状？
```

```text
肺泡蛋白质沉积症需要做哪些检查？
```

```text
统计每个科室关联的疾病数量。
```

预期结果：

- 疾病问答返回症状、检查、治疗或科室等结构化条目。
- 关系子图出现疾病节点及其关联实体。
- 统计问题返回表格或图表。
- 噪声拦截页显示按规则聚合后的数据质量问题类型，不直接暴露敏感词明细。

## 3. Nexent 智能体操作

登录 `https://nexent.mashiro.xin/` 后，可分别选择三个智能体：

| 智能体 | 用途 | 示例输入 |
| --- | --- | --- |
| 医疗数据清洗助手 | 调用 DataMate 清洗混合格式医学数据。 | `处理糖尿病任务一二三贯通演示数据集_20260627，执行任务一混合清洗，并返回质量报告和吞吐量。` |
| 医疗知识图谱问答助手 | 调用任务二流水线构建知识图谱。 | `基于糖尿病任务一清洗后的最终数据集构建知识图谱，展示实体、关系、三元组和入库结果。` |
| 医疗数据分析助手 | 查询分析库、NL2SQL 指标和可视化平台状态。 | `给出医学数据智能体可视化平台入口，并说明本轮新接入数据来源和 NL2SQL 指标。` |

智能体结果应包含工具调用、输入数据集、输出数据集或数据库、处理数量、质量指标和失败原因。若某项指标没有由工具返回，系统应明确说明该指标未提供，而不是补写推断结论。

## 4. DataMate 操作

登录 `https://datamate.mashiro.xin/` 后，可查看：

1. 数据管理中的糖尿病混合格式数据集。
2. 任务一清洗输出数据集。
3. 清洗任务状态、算子链和文件级质量报告。

任务一混合数据集应覆盖 `txt/csv/json/jsonl` 四种格式。清洗结果应保持文件格式和文件数，不应被统一转换为纯文本。

## 5. Notebook 演示

双击：

```text
demo/start_notebook_demo.bat
```

该脚本会启动 Jupyter 并打开演示 Notebook。Notebook 默认连接已经部署好的在线服务和预置演示账号，不需要用户输入密码，也不负责部署本地环境。

## 6. 常见现象

- 公网首次访问慢：等待数秒后刷新页面。
- 智能体执行时间较长：任务一和任务二会调用 DataMate 或数据库写入，耗时取决于数据量和平台状态。
- DataMate 页面为空：确认登录的是演示账号，并在数据管理中搜索“糖尿病”。
- 可视化平台数据没有变化：确认任务二流水线已成功入库并刷新分析库。
