# 任务三 NL2SQL 指标说明

任务三提供自然语言统计查询能力。系统把受支持的问题映射为只读 SQL，在 `data/task3_analytics.db` 上执行，并返回表格、图表和可追溯证据。

## 1. 指标来源

当前 NL2SQL 指标来自固定问题集的规则化评测。该路径无随机生成过程，因此同一数据库和同一问题集下可以稳定复现。

| 指标 | 当前值 | 说明 |
| --- | --- | --- |
| 通过数 | 42 / 42 | 固定问题集中全部问题返回有效 SQL 和结果。 |
| 准确率 | 100.0% | 以固定问题集为基准。 |
| 展示阈值 | 85% | 可视化平台中用于标识当前能力是否达到展示要求。 |

## 2. 支持的问题类型

| 类型 | 示例 |
| --- | --- |
| 疾病数量统计 | `统计每个科室关联的疾病数量` |
| 症状频次统计 | `统计症状出现频率最高的前 20 个症状` |
| 药物关联查询 | `阿莫西林可以治疗哪些疾病` |
| 疾病详情查询 | `糖尿病有哪些并发症` |
| 排序与 TopN | `内科最常见的 10 种疾病是什么` |

## 3. 执行边界

- 只允许执行只读查询。
- 不执行 `INSERT`、`UPDATE`、`DELETE`、`DROP` 等写操作。
- 无法识别的问题会返回失败原因，不会执行猜测性 SQL。
- 统计结果来自当前分析库，不代表医学权威结论。

## 4. 可视化展示

可视化平台会展示：

- NL2SQL 准确率；
- 通过数和总题数；
- 查询结果表；
- 可生成图表的问题类型。
