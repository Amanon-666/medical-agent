# clients 模块说明

`clients/` 封装外部平台的 HTTP 访问逻辑，使业务代码不直接拼接 DataMate 或 Nexent 接口细节。

## 模块职责

| 文件 | 职责 |
| --- | --- |
| `datamate_client.py` | 封装 DataMate 数据集、文件和任务相关接口。 |
| `nexent_client.py` | 封装 Nexent 登录、Agent、MCP 服务和工具绑定相关接口。 |

## 设计边界

- 本目录只负责平台通信、请求参数整理、响应解析和错误透传。
- 任务编排逻辑不放在这里，任务一编排位于 `mcp_server/task1/`，任务二编排位于 `mcp_server/task2/`。
- 账号、密码、服务地址从环境变量或配置文件读取，不在代码中硬编码。
- 如果 DataMate 或 Nexent 上游接口发生变化，应优先在客户端层适配，避免修改上层任务逻辑。

## 与三项任务的关系

- 任务一通过 DataMate 客户端注册和查询混合格式数据集。
- 任务二通过客户端读取任务一输出数据，为知识图谱流水线提供来源文件。
- 任务三通过 Nexent 客户端确认智能体和 MCP 工具状态，并向用户返回可视化入口。
